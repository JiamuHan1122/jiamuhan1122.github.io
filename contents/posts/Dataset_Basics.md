# 📊 Dataset Basics（数据集基础知识）

## 1. 为什么需要数据集？

在机器学习 / 深度学习中，模型本质是在做一件事：

👉 **从数据中学习规律，然后用于预测新数据**

比如：

- 图像分类：看很多猫狗图片 → 学会区分猫和狗
- NLP：看很多句子 → 学会翻译 / 分类
- 视频理解：看很多视频 → 学会识别动作

📌 **数据 = 模型的“老师”**

------

## 2. 数据集的基本划分

最核心的概念：

> **训练集 / 验证集 / 测试集**

------

### 2.1 训练集（Training Set）

👉 用来“学习”的数据

模型通过训练集不断调整参数：

```text
输入 → 模型 → 预测 → 和真实值对比 → 更新参数
```

📌 举例：

你要训练一个猫狗分类器：

- 训练集：10000张图片（猫 + 狗）
- 模型通过这些图片学习特征（耳朵、形状、纹理等）

------

### 2.2 验证集（Validation Set）

👉 用来“调参”的数据（但不参与训练）

作用：

- 判断模型有没有过拟合
- 调整超参数（learning rate、batch size等）

📌 举例：

你尝试不同学习率：

- lr = 0.1 → 验证集准确率 70%
- lr = 0.01 → 验证集准确率 85%

👉 选 0.01

------

### 2.3 测试集（Test Set）

👉 最终“考试”的数据

特点：

- **绝对不能参与训练或调参**
- 用来评估模型真实能力

📌 举例：

模型在训练 + 验证后：

- 在测试集上 accuracy = 83%

👉 这才是你最终报告的结果

------

### 🎯 一个形象类比

| 数据集 | 类比        |
| ------ | ----------- |
| 训练集 | 课本 + 作业 |
| 验证集 | 模拟考试    |
| 测试集 | 高考        |

👉 **不能用高考题来练习，否则成绩没意义**

------

## 3. 数据划分比例

常见划分方式：

```text
70% 训练 / 15% 验证 / 15% 测试
```

或者：

```text
80% 训练 / 10% 验证 / 10% 测试
```

📌 小数据集时：

👉 可以用 **交叉验证（Cross Validation）**

------

## 4. 训练过程（Training Pipeline）

一个标准训练流程如下：

------

### Step 1：准备数据

- 数据清洗
- 标注（label）
- 划分 train / val / test

------

### Step 2：模型前向传播（Forward）

```text
输入数据 → 模型 → 输出预测
```

------

### Step 3：计算损失（Loss）

 **👉损失函数是连接“模型预测”和“真实数据”的桥梁，是模型学习的唯一信号。**

 **损失函数 = 衡量“模型预测有多错”的函数**

为什么一定要有 Loss？模型训练本质是：不断调整参数 → 让 loss 最小

常见：

- 分类：Cross Entropy
- 回归：MSE

```text
Loss = 预测值 vs 真实值 的差距
```

------

### Step 4：反向传播（Backpropagation）

👉 计算梯度，更新参数：

```text
参数 = 参数 - 学习率 × 梯度
```

------

### Step 5：循环训练（Epoch）

```text
一个 epoch = 所有训练数据过一遍
```

通常训练：

- 10 ~ 100 epochs

------

### 🔁 整体流程图（概念）

```text
数据 → 模型 → 预测 → loss → 反向传播 → 更新参数 → 预测
```

------

## 5. 常见评价指标（Metrics）

不同任务用不同指标。

------

### 5.1 分类任务

------

#### ✅ Accuracy（准确率）

```text
Accuracy = 正确预测数 / 总数
```

📌 举例：

100张图，预测对80张：

👉 Accuracy = 80%

------

#### ✅ Precision / Recall / F1

用于不平衡数据（很重要！）

------

##### Precision（精确率）

```text
预测为正的中，有多少是真的
```

👉 “我说是猫的，有多少真的是猫”

------

##### Recall（召回率）

```text
真实为正的中，被预测出来多少
```

👉 “所有猫中，我找出来多少”

------

##### F1-score

```text
F1 = Precision 和 Recall 的平衡
```

------

### 🎯 举个例子（很重要）

假设：

- 100张图片
- 只有10张是猫

模型全预测“不是猫”：

👉 Accuracy = 90% 😅（很高）

但：

- Recall = 0 ❌（完全没用）

👉 所以不能只看 Accuracy

------

### 5.2 回归任务

------

#### ✅ MSE（均方误差）

```text
MSE = (预测值 - 真实值)^2 的平均
```

------

#### ✅ MAE（绝对误差）

```text
MAE = |预测 - 真实|
```

------

### 5.3 检测 / 分割（进阶）

------

#### ✅ IoU（Intersection over Union）

```text
IoU = 预测区域 ∩ GT / 预测区域 ∪ GT
```

👉 衡量框的重叠程度

------

#### ✅ mAP（mean Average Precision）

用于目标检测（YOLO / Faster R-CNN）

👉 综合 Precision-Recall

------

## 6. 过拟合 vs 欠拟合

------

### 欠拟合（Underfitting）

👉 模型太弱

表现：

- 训练集差
- 测试集也差

------

### 过拟合（Overfitting）

👉 模型记住训练数据，但不会泛化

表现：

- 训练集很好
- 测试集很差 ❌

------

### 🎯 形象理解

| 类型   | 类比     |
| ------ | -------- |
| 欠拟合 | 没学会   |
| 过拟合 | 死记硬背 |
| 正常   | 理解了   |

------

## 7. 常见提升方法

------

### 数据层面

- 数据增强（augmentation）
- 更多数据

------

### 模型层面

- 更深模型（避免欠拟合）
- 正则化（避免过拟合）

------

### 训练层面

- early stopping
- learning rate 调整

------

## 8. 一个完整小例子

假设你做一个：

👉 “猫狗分类项目”

------

### 数据：

- 10000张图
- train: 8000
- val: 1000
- test: 1000

------

### 训练：

```text
epoch 1 → loss 下降
epoch 10 → accuracy 提升
epoch 20 → val accuracy 开始下降
```

👉 出现过拟合

------

### 解决：

- 加数据增强
- 加 dropout

------

### 最终：

```text
Test Accuracy = 85%
```

------

# 🧠 一句话总结

👉 **机器学习的本质就是：用训练数据学习规律，用测试数据验证泛化能力。**
